0. Một số kiến trúc phổ biến
----------------------------------------------------------------------------------------------------------------------------------------------------------
1. Kiến trúc Microservices
------------------------------------------------------------------------------
Đặc điểm: Sử dụng kiến trúc microservices để chia nhỏ ứng dụng thành các dịch vụ nhỏ, độc lập.
Dịch vụ AWS phổ biến: 
- Amazon ECS/EKS (Chạy container), 
- AWS Lambda (Serverless), 
- Amazon API Gateway (Quản lý API), 
- Amazon SQS (Hàng đợi tin nhắn).

Lợi ích: Tăng khả năng mở rộng và cho phép các nhóm phát triển hoạt động độc lập trên các dịch vụ khác nhau.

2. Kiến trúc Serverless
------------------------------------------------------------------------------
Đặc điểm: Loại bỏ hoàn toàn việc quản lý server, tự động mở rộng và trả phí dựa trên mức sử dụng thực tế.
Dịch vụ AWS phổ biến: 
- AWS Lambda
- Amazon API Gateway
- Amazon DynamoDB
- Amazon S3
- AWS Step Functions.
Lợi ích: Giảm chi phí vận hành, tối ưu hóa quy trình phát triển và triển khai.


3. Kiến trúc Data Lake và Big Data
------------------------------------------------------------------------------
Đặc điểm: Tạo ra kho dữ liệu lớn để lưu trữ, phân tích và xử lý dữ liệu lớn.
Dịch vụ AWS phổ biến: 
- Amazon S3 (Kho lưu trữ dữ liệu), 
- AWS Glue (ETL), 
- Amazon Redshift (Kho dữ liệu), 
- Amazon Athena (Truy vấn dữ liệu), 
- Amazon EMR (Xử lý dữ liệu lớn).
Lợi ích: Cho phép phân tích dữ liệu nhanh chóng và hiệu quả từ nhiều nguồn khác nhau.


4. Kiến trúc Web Application Hosting
------------------------------------------------------------------------------
Đặc điểm: Triển khai và quản lý các ứng dụng web với tính năng mở rộng, dự phòng cao.
Dịch vụ AWS phổ biến: 
- AWS Elastic Load Balancing (Phân tải), 
- Amazon EC2 + Auto Scaling/ Lambda
- Amazon RDS (Cơ sở dữ liệu), 
- Amazon CloudFront (CDN), 
- Amazon Route 53 (DNS).
- S3 lưu trữ dữ liệu 
Lợi ích: Đảm bảo hiệu suất cao, bảo mật và khả năng mở rộng khi lưu trữ ứng dụng web.


5. Kiến trúc Disaster Recovery (Phục hồi sau thảm hoạ)
------------------------------------------------------------------------------
Đặc điểm: Tạo khả năng phục hồi và khôi phục dữ liệu nhanh chóng trong các tình huống khẩn cấp.
Dịch vụ AWS phổ biến: 
- Amazon S3 và S3 Glacier (Sao lưu dữ liệu), 
- AWS Backup, 
- Amazon RDS (Sao lưu dữ liệu), 
- Amazon Route 53 (Chuyển đổi DNS).
Lợi ích: Đảm bảo dữ liệu không bị mất mát và có thể phục hồi nhanh chóng sau các sự cố.


6. Kiến trúc IoT
------------------------------------------------------------------------------
Đặc điểm: Quản lý và xử lý dữ liệu từ các thiết bị IoT trong thời gian thực.
Dịch vụ AWS phổ biến: 
- AWS IoT Core, 
- Amazon Kinesis (Xử lý dữ liệu thời gian thực), 
- AWS Lambda, 
- Amazon S3.
Lợi ích: Tối ưu hóa quá trình giám sát và phân tích dữ liệu từ các thiết bị IoT.


7. Kiến trúc Machine Learning và AI
------------------------------------------------------------------------------
Đặc điểm: Triển khai các mô hình machine learning và AI để phân tích, dự đoán dữ liệu.
Dịch vụ AWS phổ biến: 
- Amazon SageMaker (Xây dựng và triển khai mô hình), 
- AWS Lambda (Serverless),
- Amazon S3 (Lưu trữ dữ liệu).
Lợi ích: Cho phép xây dựng, huấn luyện và triển khai các mô hình ML với chi phí và thời gian tối ưu.
Mỗi kiến trúc sẽ phù hợp với các nhu cầu khác nhau tùy vào yêu cầu và quy mô của dự án.


8. Kết nối với database RDS
------------------------------------------------------------------------------
Dùng kết hợp RDS Proxy và SQS là một giải pháp khả thi và có thể đem lại hiệu quả cao, đặc biệt trong môi trường có lưu lượng biến động hoặc có thể gặp vấn đề với các kết nối đồng thời đến cơ sở dữ liệu.
Cách kết hợp RDS Proxy và SQS:
- RDS Proxy giúp quản lý kết nối cơ sở dữ liệu hiệu quả hơn bằng cách duy trì các kết nối tồn tại và tái sử dụng chúng, thay vì phải mở một kết nối mới cho mỗi yêu cầu từ Lambda. Điều này giúp giảm tải cho cơ sở dữ liệu, tránh tình trạng lỗi timeout khi quá nhiều Lambda function cố gắng kết nối cùng một lúc.
- SQS có thể giúp kiểm soát tốc độ gửi yêu cầu từ Lambda đến cơ sở dữ liệu. Thay vì Lambda function gửi yêu cầu trực tiếp vào cơ sở dữ liệu, yêu cầu sẽ được đẩy vào hàng đợi (queue) SQS. Một hoặc nhiều Lambda functions sau đó sẽ xử lý các tin nhắn từ SQS theo từng đợt (batch), giúp giảm tải lên cơ sở dữ liệu và tránh quá tải kết nối vào một thời điểm.



I. Admin Service 
----------------------------------------------------------------------------------------------------------------------------------------------------------
1. Phân quyền 
------------------------------------------------------------------------------
Phân cấp bậc tài khoản trong AWS
Organization (Root)
  ├── Organizational Units (OUs)
  │       ├── Member Account 1
  │       ├── Member Account 2
  │       └── Member Account 3
  │
  └── Member Account 4
           ├── IAM Groups (Admins, Developers)
           │       ├── IAM User 1
           │       └── IAM User 2
           │
           ├── IAM Role (Cross-account, EC2 Role)
           └── Resources (EC2, S3, RDS)

1 tổ chức có nhiều account
1 account có nhiều user,group,role, resource


IAM(Identity and Access Management)
- Dịch vụ chính cho phép quản lý tk trên aws và phân quyền
- Users: 
	- Đại diện cho người dùng với thông tin đăng nhập cụ thể.
	- User: A, B C
- Groups: 
	- Tập hợp các Users có cùng quyền.
	- Ví dụ: group dev, prod ...
- Policies: Định nghĩa các quyền truy cập cụ thể bằng các JSON document
	- Ttập hợp các quy tắc xác định quyền cho phép hoặc từ chối hành động trên các tài nguyên AWS. 
	- Chính sách có thể được áp dụng cho người dùng (IAM users), nhóm (groups), hoặc vai trò (roles) để kiểm soát những gì họ có thể làm trong tài khoản AWS.
- Roles: Các quyền tạm thời cho phép các thực thể hoặc dịch vụ giả danh để truy cập tài nguyên mà không cần thông tin đăng nhập.
	- Thực thể IAM được tạo ra để gán các quyền thông qua policies. 
	- Role tương tự như người dùng IAM, nhưng thay vì gán cho một cá nhân hoặc nhóm người dùng, role được gán cho các thực thể AWS khác 
	- Ví dụ:(như EC2 instance, Lambda function) hoặc cho phép người dùng/ứng dụng bên ngoài (có quyền tạm thời) sử dụng.
- Security: MFA + Password Policy
- Audit: IAM Credential Reports & IAM Access Advisor


AWS IAM Identity Center:
- Dịch vụ giúp quản lý truy cập và danh tính người dùng, tạo SSO tiện lợi cho người dùng truy cập vào nhiều tài khoản AWS và ứng dụng.
- Multi-Account Permissions
	- Manage access across AWS accounts in your AWS Organization
	- Permission Sets – a collection of one or more IAM Policies assigned to users and groups to define AWS access
- Application Assignments
	- SSO access to many SAML 2.0 business applications (Salesforce, Box, Microsoft 365, …)
	- Provide required URLs, certificates, and metadata
- Attribute-Based Access Control (ABAC)
	- Fine-grained permissions based on users’ attributes stored in IAM Identity Center Identity Store
	- Example: cost center, title, locale, …
	- Use case: Define permissions once, then modify AWS access by changing the attributes


AWS Organizations:
- Dịch vụ nền tảng cho việc quản lý nhiều tài khoản AWS
- Khi khởi tạo sẽ cần một tài khoản đóng 2 vài trò chính:
	- Management account: tài khoản quản trị, có quyền trên tất cả các tk khác
	- Payer account: tài khoản thanh toán
- Cho phép bạn tổ chức các tài khoản vào các nhóm và áp dụng chính sách quản trị cho từng nhóm thông qua Service Control Policies (SCPs)
- Tính năng chính của AWS Organizations:
	- Quản lý đa tài khoản: Tạo, quản lý và tập hợp các tài khoản AWS dưới một tổ chức duy nhất.
	- Service Control Policies (SCPs): Chính sách kiểm soát dịch vụ cho phép bạn áp dụng các quy tắc và giới hạn quyền truy cập đối với các dịch vụ AWS trên nhiều tài khoản.
	- Tạo tài khoản AWS: Bạn có thể tự tạo tài khoản mới trong tổ chức và quản lý quyền truy cập giữa các tài khoản.
	- Consolidated Billing: Tập hợp hoá đơn của tất cả các tài khoản trong một tổ chức, giúp dễ dàng quản lý chi phí.
- Use case:
	- Create a service control policy (SCP) to prevent tag modi¬cation except by authorized principals


AWS Control Tower
- Dịch vụ cao cấp hơn, được xây dựng dựa trên AWS Organizations để giúp người dùng dễ dàng thiết lập và quản lý môi trường AWS đa tài khoản với các chuẩn mực về bảo mật và quản trị. 
- Control Tower tự động hoá nhiều quy trình thiết lập và quản lý,  phù hợp với những doanh nghiệp muốn một môi trường được quản lý và bảo vệ theo các best practices ngay từ đầu 
- Tính năng chính của AWS Control Tower:
	- Landing Zone: Tự động tạo và quản lý một môi trường AWS đa tài khoản với cấu hình sẵn cho bảo mật và quản trị.
	- Guardrails: Các chính sách bảo vệ được thiết lập sẵn (preventive và detective guardrails) giúp kiểm soát việc tuân thủ các quy định bảo mật và quản trị.
	- Account Factory: Tự động hóa việc tạo tài khoản AWS mới theo các cấu hình bảo mật và quản trị đã thiết lập sẵn.
	- Tự động hóa và tích hợp: Control Tower tích hợp các dịch vụ như AWS Organizations, AWS SSO, và CloudTrail để cung cấp một giải pháp quản lý tài khoản toàn diện.
	- Quản lý tập trung: Control Tower cung cấp giao diện quản lý đơn giản để theo dõi trạng thái tuân thủ của tất cả các tài khoản.

AWS Directory Services
- AWS Managed Microsoft AD
	- Create your own AD in AWS, manage users locally, supports MFA
	- Establish "trust" connections with your onpremises AD
- AD Connector
	- Directory Gateway (proxy) to redirect to onpremises AD, supports MFA
	- Users are managed on the on-premises AD
- Simple AD
	- AD-compatible managed directory on AWS
	- Cannot be joined with on-premises AD
- Use case:

2. Quản lý chi phí
------------------------------------------------------------------------------
AWS Cost Explorer
- Tập trung vào phân tích và hình dung chi phí, giúp tối ưu hóa chi tiêu.
- Visualize, understand, and manage your AWS costs and usage over time
- Create custom reports that analyze cost and usage data.
- Analyze your data at a high level: total costs and usage across all accounts
- Or Monthly, hourly, resource level granularity
- Choose an optimal Savings Plan (to lower prices on your bill)
- Forecast usage up to 12 months based on previous usage
- Use Cases:
	- Visualizing past costs and usage patterns.
	- Identifying cost allocation trends and usage spikes.
	- Making decisions based on historical data to optimize your AWS spend.



AWS budget
- Mục đích chính: Giúp theo dõi và thiết lập cảnh báo chi phí hoặc sử dụng dịch vụ theo ngân sách.
- Chức năng:
	- Cho phép đặt ngân sách chi phí hoặc mức sử dụng dựa trên dự đoán hoặc ngân sách cố định.
	- Tạo cảnh báo qua email hoặc SNS khi chi phí/ mức sử dụng đạt đến các ngưỡng đặt trước (ví dụ: 50%, 80%, 100% của ngân sách).
	- Có thể áp dụng ngân sách theo từng dịch vụ, tài khoản, khu vực, hoặc theo các tiêu chí khác như thẻ.
- Use cases:
	- Monitoring and controlling spending by setting limits.
	- Receiving notifications when spending exceeds set limits.
	- Proactively managing and preventing overspending.


AWS Cost Anomaly Detection:
- Công cụ trong AWS Billing and Cost Management được thiết kế để phát hiện và cảnh báo về các chi phí bất thường.
- Cách hoạt động:
	- Tạo Monitor: Giám sát một nhóm tài nguyên hoặc dịch vụ AWS cụ thể để theo dõi chi phí.
	- Tạo Anomaly Detection Subscriptions: Gửi thông báo cho các bên liên quan qua Amazon SNS khi phát hiện các mẫu chi phí bất thường.
	- Phân tích sự bất thường: Giúp bạn hiểu nguyên nhân chi phí gia tăng bất thường, từ đó hành động nhanh chóng để kiểm soát chi phí.


II. System
----------------------------------------------------------------------------------------------------------------------------------------------------------
1. Server (EC2)
------------------------------------------------------------------------------
Server trong AWS được chia làm 2 loại:
- Amazon EC2 Instance:
	- EC2 Instance là một phiên bản ảo của máy chủ trên AWS. 
	- Đây là một đơn vị tài nguyên tính toán độc lập chạy trên cơ sở hạ tầng ảo của AWS, nơi mà bạn có thể chạy ứng dụng, cài đặt phần mềm, và thực hiện các tác vụ tính toán.
	- Sử dụng vCPU ( tương đương thread của máy chủ vật lý)
	- Không liên quan tới license của các phần mềm đang chạy
- Amazon EC2 Dedicated Host:
	- Máy chủ vật lý thực sự, dành riêng cho doanh nghiệp
	- Trên các host có thể chạy các EC2 instances.
	- Có thể sử dụng để chạy các phền mềm có license tính theo core hoặc socket


EC2(Elastic Compute Cloud): server ảo hóa của AWS 
- Bao gồm:
	- Tài nguyên xử lý; CPU, Memory
	- Tài nguyên lưu trữ: disk (thường sử dụng EBS) chứa dữ liệu ( application, DB ...)
- AMI (Amazon Machine Image) are a customization of an EC2 instance, 
	- Được sử dụng để tạo ra các server EC2 mới một cách nhanh chóng
	- Cách tạo ta AMI:
		- Từ một Instance Đang Chạy ()
		- Từ các bản snapshot( backup) của máy chủ EC2
		- Sử dụng Amazon Data Lifecycle Manager (DLM)
		- Từ các bản snapshot của disk (EBS), restore và gắn snapshot vào EC2 để chạy
		- Sử dụng AWS Marketplace hoặc Chọn AMI Sẵn có
	- Note:
		- Permissions: AMI có thể được chia sẻ giữa các tài khoản AWS, hoặc có thể giới hạn chỉ cho phép truy cập bởi chủ sở hữu.
		- Region: AMI chỉ được tạo và sử dụng trong một vùng (region) cụ thể. Nếu muôn sử dụng ở Region khác thì cần copy AMI giữa các vùng.
- Data Lifecycle Manager
	- Dịch vụ tự động hóa quá trình quản lý vòng đời của snapshot EBS và Amazon Machine Images (AMIs)
	- Quản lý Snapshot tự động:
	- Chính sách lưu trữ
	- Use case:
		- Sao lưu tự động: Bạn có thể sử dụng DLM để tạo snapshot tự động cho các ổ EBS đang được sử dụng bởi các instance EC2 để bảo vệ dữ liệu và phục hồi dễ dàng trong trường hợp gặp sự cố.
		- Tuân thủ và báo cáo: DLM giúp đảm bảo rằng các snapshot cũ sẽ bị xóa tự động theo các chính sách tuân thủ mà bạn đặt ra, đồng thời giúp dễ dàng báo cáo về việc sao lưu dữ liệu.
		- Quản lý chi phí lưu trữ: Bằng cách tự động xóa các snapshot không cần thiết, DLM giúp giảm chi phí lưu trữ dài hạn của bạn.
	
	
EC2 Instances Purchasing Options

- Reserved Instances – long workloads (1 & 3 years)
	- Cam kết sử dụng một instance cụ thể trong một availability zone trong 1 hoặc 3 năm.
- Convertible Reserved Instances – long workloads with flexible instances
- Phân loại instance theo cấu hình:
	- General Purpose: M5, T3 phù hợp cho hầu hết các ứng dụng.
	- Compute Optimized: C5 cho các ứng dụng yêu cầu CPU mạnh.
	- Memory Optimized: R5, X1e cho các ứng dụng yêu cầu bộ nhớ lớn.
	- Storage Optimized: I3, D2 cho các ứng dụng lưu trữ dữ liệu lớn.
	- GPU: P3, G4dn cho các ứng dụng AI, đồ họa.
	- HPC: HPC6id cho các tính toán khoa học và HPC.
	- Burstable Performance Instances (Hiệu suất burstable)
- Phân loại instance theo nhu câu:
	- On-Demand Instances 
		- Cung cấp máy chủ theo yêu cầu, trả theo thời gian sử dụng
		– short workload, predictable pricing, pay by second -> trả theo thời gian sử dụng (khá đắt)
	- Reserved Instances: 
		- Đặt trước instance từ 1-3 năm để giảm 30-70% chi phí
		- Capacity Reservations – reserve capacity in a specific AZ for any duration
	- Spot Instances 
		- Loại Instance rẻ nhất nhưng có thể bị terminate bất cứ lúc nào

	- Savings Plans (1 & 3 years) –commitment to an amount of usage, long workload 	-> rẻ, phù hợp khi có kế hoạch dài hạn
		- Compute Savings Plan:

			- Lợi ích: Tiết kiệm chi phí cao nhất và linh hoạt nhất trong việc thay đổi cấu hình dịch vụ.

			- Hạn chế: Phải cam kết sử dụng một loại instance cụ thể, không thể thay đổi giữa các loại EC2 instance families (ví dụ, từ t3 sang m5).
		- EC2 Instance Savings Plan (Convertible):

			- Có thể chuyển đổi giữa các loại instance khác nhau, nhưng vẫn phải cam kết với một khu vực và cấu hình tương tự 
			- Linh hoạt hơn so với loại EC2 Instance Savings Plan cố định, nhưng vẫn hạn chế hơn so với Compute Savings Plan.
- Optional:

- Monitor:

	- Monitor đơn gian: Use Amazon CloudWatch built-in EC2 memory metrics to track the application performance for future capacity planning

EC2 Placement Strategies
- Chiến lược cho việc khởi động các instance EC2
- Bao gồm 3 chiến lược chính:
	- Cluster: Tất cả các instance trong cùng một placement group được triển khai trên cùng một Availability Zone và trên cùng một hoặc một số ít máy chủ vật lý.
		- Ưu điểm: Tối ưu hóa băng thông mạng giữa các instance do chúng nằm gần nhau Giảm độ trễ khi giao tiếp giữa các instance.
		- Nhược điểm: Nếu máy chủ vật lý gặp sự cố, tất cả các instance trong nhóm cluster sẽ bị ảnh hưởng, gây ra thời gian ngừng hoạt động cho toàn bộ ứng dụng.
		- Sử dụng: Phù hợp cho các ứng dụng cần băng thông mạng cao và không cần tính sẵn có cao.
	- Spread: Các instance được triển khai trên các máy chủ vật lý khác nhau, giúp phân tán rủi ro. Mỗi instance sẽ được triển khai trên một máy chủ vật lý duy nhất.
		- Ưu điểm: 
			- Tăng tính khả dụng cho ứng dụng vì một sự cố ở một máy chủ vật lý không ảnh hưởng đến các instance khác.
			- Giảm thiểu rủi ro từ lỗi phần cứng vì các instance không nằm trên cùng một máy chủ.
		- Nhược điểm: Có thể không tối ưu về mặt băng thông giữa các instance do chúng có thể nằm xa nhau về mặt vật lý.
	- Partition: Mô tả: Các instance được phân phối thành các partitions. Mỗi partition sẽ chứa một số instance, và các instance trong một partition sẽ nằm trên các máy chủ vật lý khác nhau.
		- Ưu điểm:
			- Cung cấp khả năng sẵn có cao hơn so với cluster strategy vì ngay cả khi một partition bị ảnh hưởng, các partition khác vẫn hoạt động bình thường.
			- Tối ưu hóa hiệu suất mạng giữa các instance trong cùng một partition, đồng thời giữ tính khả dụng khi có lỗi xảy ra.
		- Nhược điểm:
			- Phức tạp hơn trong việc quản lý so với cluster và spread.


2. Storage
------------------------------------------------------------------------------
Storage concept:
- Storage là nơi lưu trữ dữ liệu
- Hiệu năng của storage có thể được đo bởi:
	- IOPS (Input/Output Operations Per Second)
		- Số lượng các thao tác nhập/xuất dữ liệu (input/output operations) mà một thiết bị lưu trữ có thể thực hiện trong một giây.
	- MBps (Megabytes per second)là tốc độ truyền dữ liệu tính bằng số megabyte mỗi giây. 
		- Đây là thông số đo tốc độ mà dữ liệu có thể được đọc từ hoặc ghi vào một thiết bị lưu trữ trong một giây.


### Block Storage (Lưu trữ khối)
Concept: 
- Block Storage là lớp lưu trữ cơ bản và linh hoạt nhất. 
- Nó chia dữ liệu thành các khối (blocks) riêng lẻ, mỗi khối có một địa chỉ duy nhất. 
- Các khối này được lưu trữ và quản lý độc lập, và hệ thống lưu trữ không biết về cấu trúc của dữ liệu trong từng khối. 
- Điều này giúp block storage có hiệu suất cao, nhưng để sử dụng dữ liệu, cần một hệ thống quản lý tập tin (file system) để tổ chức và gắn kết các khối dữ liệu này.
- Trong AWS Block Storage bao gồm:
	- EBS: (Elastic Block Store Volume) 
	- EC2 Instance Store


EBS: (Elastic Block Store Volume) 
- Lưu trữ khối, phù hợp cho hệ thống cần truy xuất nhanh, hiệu suất cao
- Thường dùng để cài hệ điều hành, cơ sở dữ liệu.
- Optional: 
	- Enable EBS encryption by default in the AWS Region where the EKS cluster will be created. Select the customer managed key as the default key.
	- Create the EBS volumes as encrypted volumes. Attach the EBS volumes to the EC2 instances.
	- Lock the EBS snapshots to prevent deletion.
- Phân loại:
	- General Purpose SSD - gp3 - 16,000 - 1,000 MB/giây	Workload phổ biến (có thể điều chỉnh độc lập với dung lượng).
	- General Purpose SSD - gp2 - 16,000 - 250 MB/giây		Hệ điều hành, DB nhỏ
	- Provisioned IOPS SSD - io2 - 64,000-256,000 -	4,000 MB/giây	Workload cần IOPS cao và ổn định
		- Cho phép Multi-Attach.
	- Provisioned IOPS SSD - io1 - 64,000 -	1,000 MB/giây	DB quan trọng, yêu cầu độ trễ thấp
		- Cho phép Multi-Attach.
	- Throughput Optimized HDD	st1	500	500 MB/giây	Ứng dụng phân tích, xử lý dữ liệu lớn
	- Cold HDD	sc1	250	250 MB/giây	Lưu trữ dữ liệu ít truy cập


EC2 Instance Store
- Lưu trữ Tạm Thời: Dữ liệu được lưu trữ sẽ mất khi máy chủ EC2 bị dừng, khởi động lại hoặc gặp sự cố. 
- Hiệu năng Cao: rất phù hợp cho các ứng dụng yêu cầu tốc độ truy cập nhanh như xử lý video, cơ sở dữ liệu tạm thời, hoặc các tác vụ tính toán nặng.
- Khả Năng Mở Rộng: 
	- Số lượng và kích thước của instance store phụ thuộc vào loại phiên bản của máy chủ EC2 
	- Một số loại phiên bản máy chủ EC2 cung cấp nhiều dung lượng instance store hơn những loại khác.
- Chi Phí: Chi phí sử dụng thường thấp hơn so với các dịch vụ lưu trữ khác, nhưng cần xem xét rằng dữ liệu không được bảo toàn khi máy chủ bị dừng hoặc gặp sự cố.
- Use case:
	- Tác vụ Tạm Thời: Khi bạn cần lưu trữ dữ liệu tạm thời trong quá trình xử lý mà không cần phải giữ lại lâu dài.
	- Ứng dụng Yêu cầu Hiệu suất Cao: Như cơ sở dữ liệu hoặc ứng dụng xử lý lớn cần truy cập dữ liệu với tốc độ cao.
	- Chi phí Thấp: Khi bạn muốn giảm chi phí lưu trữ cho các tác vụ tạm thời.
	
	
### File Storage (Lưu trữ tệp)
Concept: 
- File Storage là tầng trên của block storage 
- Sử dụng một hệ thống tập tin (file system) để sắp xếp và quản lý các khối dữ liệu thành các thư mục và tập tin mà người dùng hoặc ứng dụng có thể dễ dàng truy cập.
- File storage giúp quản lý dữ liệu theo dạng tệp (files) và cho phép truy cập thông qua các giao thức phổ biến như NFS hoặc SMB.
- Trong AWS, File Storage bao gồm:
	- EFS: (Elastic File System)
	- FSx


EFS: (Elastic File System)
- Lưu trữ file tập trung, cho phép nhiều máy chủ EC2 truy cập cùng lúc
- Thích hợp cho các ứng dụng cần chia sẻ tệp giữa nhiều máy.
- Chỉ sử dụng với các server linux


FSx: 
- Dịch vụ lưu trữ đám mây được quản lý hoàn toàn, cung cấp các hệ thống file systems cho các ứng dụng yêu cầu lưu trữ tệp phân tán và hiệu suất cao. 
- Amazon FSx cung cấp các loại file system khác nhau, giúp hỗ trợ các ứng dụng yêu cầu lưu trữ tệp với các tính năng đặc thù.
- Bao gồm
	- FSx for Windows File Server 
		- Giải pháp sharefile giữa các Window server
		- Giao thức SMB
		- Có thể join to AD on premises to restrict access
		- Không phù hợp vs container
	- FSx for Lustre
		- Giải pháp sharefile giữa các máy chủ > for HPC (High Performance Computing)
		- Phù hợp vs container
	- FSx for NetApp ONTAP
		- Dịch vụ lưu trữ tệp được quản lý hoàn toàn (fully managed) trên AWS, dựa trên công nghệ ONTAP của NetApp
		- Hỗ trợ giao thức đa dạng:
			- NFS (Network File System): Dành cho Linux và ứng dụng containerized.
			- SMB (Server Message Block): Phù hợp với ứng dụng Windows.
			- iSCSI: Dùng cho workloads dạng block storage.
		- HA:
			- Create an FSx for ONTAP instance in the secondary Region. Use NetApp SnapMirror to replicate data from the primary Region to the
secondary Region.
	- FSx for OpenZFS
		- Dịch vụ lưu trữ tệp được quản lý hoàn toàn (fully managed) dựa trên OpenZFS
		- Hiệu năng cao
		- Use case:
			- Phân tích dữ liệu và Machine Learning:
            - Ứng dụng containerized:
            - Hệ thống phát triển phần mềm:
            - Xử lý media:

	
### Object Storage (Lưu trữ đối tượng)
Concept: 
- Object storage lưu trữ dữ liệu dưới dạng các đối tượng (objects). 
- Object Storage là tầng trên của block storage 
- Object Storage gồm nhiều bucket, trong môi bucket là một không gian phẳng (flat structure)
- Mỗi đối tượng bao gồm 
	- Dữ liệu (content)
	- Metadata (thông tin mô tả dữ liệu)
	- Object key: tên định danh duy nhất của đối tượng trong bucket. Ví dụ: object key với tên đầy đủ images/2023/photo.jpg.
	- Version ID (optional): các phiên bản của đối tượng ( khi bật tính năng versioning) 
- Trong AWS, File Storage bao gồm:
	- S3


S3: 
- Lưu trữ đối tượng, phù hợp cho lưu trữ dữ liệu lớn, không cần xử lý trực tiếp trên hệ điều hành (như hình ảnh, video, backup). 
- Chia thành nhiều bucket, mỗi bucket là một không gian phẳng chứa các đối tượng 
- Mối đối tượng có thể được gán các loại lưu trữ khác nhau, lựa chọn loại lưu trữ trong S3 cho từng mục đích sử dụng,
- Có thể setup S3 Lifecycle Policies để để tự động di chuyển các đối tượng giữa các loại lưu trữ khác nhau trong cùng bucket
- Phân loại:
	- S3 Standard
		- Đặc điểm: Dành cho các dữ liệu truy cập thường xuyên.
		- Chi phí: Cao hơn các lớp khác vì tốc độ truy cập nhanh và khả năng chịu lỗi cao.
		- Sử dụng phù hợp cho: Dữ liệu quan trọng, cần truy cập nhanh và thường xuyên như nội dung trang web, tệp ứng dụng.
	- S3 Intelligent-Tiering
		- Đặc điểm: Tự động chuyển đổi giữa các cấp lưu trữ dựa trên tần suất truy cập.
		- Chi phí: Phí quản lý nhỏ cho tính năng tự động chuyển cấp, nhưng tối ưu chi phí khi dữ liệu không được truy cập thường xuyên.
		- Sử dụng phù hợp cho: Dữ liệu có mẫu truy cập không đều đặn hoặc khó dự đoán.
	- S3 Standard-IA (Infrequent Access)
		- Đặc điểm: Lưu trữ cho dữ liệu ít truy cập nhưng cần có sẵn khi cần.
		- Chi phí: Thấp hơn S3 Standard, nhưng có phí truy xuất (retrieval fee) khi tải dữ liệu.
		- Sử dụng phù hợp cho: Dữ liệu cần giữ lâu dài nhưng ít truy cập như bản sao lưu, dữ liệu bảo trì.
	- S3 One Zone-IA
		- Đặc điểm: Lưu trữ cho dữ liệu ít truy cập nhưng chỉ trong một khu vực (availability zone).
		- Chi phí: Rẻ hơn Standard-IA vì không có khả năng chịu lỗi khu vực.
		- Sử dụng phù hợp cho: Dữ liệu sao lưu ít quan trọng, dễ thay thế hoặc dữ liệu không đòi hỏi khả năng chịu lỗi cao.
	- S3 Glacier
		- Đặc điểm: Lưu trữ lâu dài cho dữ liệu không truy cập thường xuyên, thời gian truy xuất từ vài phút đến vài giờ.
		- Chi phí: Rất thấp, phù hợp cho lưu trữ dữ liệu dài hạn.
		- Sử dụng phù hợp cho: Lưu trữ dữ liệu lưu trữ dài hạn, lưu trữ hồ sơ, tài liệu pháp lý.
		- Khả năng lấy dữ liệu:
			- Sử dụng Expedited nếu cần truy xuất nhanh (1-5 phút).
			- Sử dụng Standard cho nhu cầu khôi phục thông thường. (3-5 giờ)
			- Sử dụng Bulk khi cần khôi phục dữ liệu lớn với chi phí thấp nhất (5-12 giờ)
	- S3 Glacier Deep Archive
		- Đặc điểm: Lưu trữ lâu dài cho dữ liệu rất ít khi truy cập, thời gian truy xuất từ 12 đến 48 giờ.
		- Chi phí: Thấp nhất trong các loại S3.
		- Sử dụng phù hợp cho: Lưu trữ dữ liệu lưu trữ dài hạn không yêu cầu truy cập thường xuyên, phù hợp cho tuân thủ và lưu trữ hồ sơ.
	- S3 Outposts
		- Đặc điểm: Cung cấp lưu trữ S3 tại chỗ trong các trung tâm dữ liệu tại cơ sở của doanh nghiệp (on-premises).
		- Chi phí: Được tính riêng cho Outposts.
		- Sử dụng phù hợp cho: Dữ liệu cần lưu trữ tại cơ sở vì lý do quy định, độ trễ thấp hoặc không thể đưa lên đám mây.
- How to connect (recommend): 
	- Create a gateway VPC endpoint for Amazon S3 (in same VPC)
	- Create a interface VPC endpoint for Amazon S3 (in another location)
- Optional
	- Mã hóa: SSE-S3: mã hóa đơn giản, chi phí rẻ, nhưng ko có các tính năng cao cấp như xoay vòng khóa ...
	- Requester Pays : người gọi dữ liệu tự trả tiền (ng sở hữu ko mất tiền)
	- Object Lock trên Amazon S3 cung cấp các chế độ khóa giúp bảo vệ dữ liệu chống lại việc xóa hoặc thay đổi ngẫu nhiên
		- S3 Object Lock ở chế độ compliance -> chế độ nghiêm ngặt nhất, không thể thay đổi
		- S3 Object Lock ở chế độ governance -> chế độ nghiêm ngặt với user thường, admin vẫn có thể thay đổi
	- Có thể sử dụng Signed URLs để bảo mật và kiểm soát truy cập
	- Phân quyền cho ứng dụng: Create dedicated S3 access points and access point policies for each application
	- S3 Block Public Access là một tính năng của Amazon S3 giúp bảo vệ dữ liệu trong các bucket và object bằng cách ngăn chặn các quyền truy cập công khai ngoài ý muốn:
		- Cấp tài khoản (account level)
		- Cấp bucket (bucket level)
	- Share data: Cross-account access là cách tiếp cận tối ưu để chia sẻ dữ liệu giữa các tài khoản AWS.
	- Increate upload and download performance:
		- Upload: Use S3 multipart uploads.
		- Download: Fetch multiple byte-ranges of an object in parallel
- Use case: 
	- Lưu trữ dữ liệu dài hạn với chi phí rẻ
	- Xây dựng website tĩnh: 
		- Sử dụng kèm Cloud Front: Host web tĩnh (chứa HTML, CSS, JavaScript và hình ảnh)
		- Create an origin access identity (OAI). 
		- Assign the OAI to the CloudFront distribution.

### Hybrid Storage: 
- Kết hợp giữa lưu trữ đám mây và on-premises
- Dịch vụ trên AWS: Storage Gateway
- Bridge giữa dữ liệu on-premises và dữ liệu đám mây
- Cho phép doanh nghiệp sử dụng AWS như một giải pháp lưu trữ có thể mở rộng mà không cần thay đổi ứng dụng hoặc quy trình làm việc hiện tại
- Bao gồm 3 loại chính:
	- File Gateway:
		- Cho phép người dùng lưu trữ và truy cập dữ liệu tệp trên Amazon S3 dưới dạng objects thông qua giao thức SMB (Server Message Block) và NFS (Network File System).
		- Có thể gắn với: Amazon S3
	- Volume Gateway:
		- Cached Volumes: Dữ liệu chính được lưu trữ trong Amazon S3, chỉ một phần dữ liệu thường xuyên truy cập được lưu cục bộ để giảm độ trễ.
		- Stored Volumes: Dữ liệu chính được lưu trữ tại chỗ và đồng bộ hóa bản sao với Amazon S3 để có khả năng khôi phục.
		- Có thể gắn với: Amazon S3 (với khả năng tạo snapshot để phục hồi qua EBS)
	- Tape Gateway: 
		- Cho phép các ứng dụng sao lưu sử dụng băng từ ảo để lưu trữ dữ liệu trên Amazon S3 và Amazon S3 Glacier (Glacier hoặc Glacier Deep Archive).
		- Có thể gắn với: Amazon S3, S3 Glacier
	- FSx File Gateway: 
		- Cung cấp khả năng truy cập từ on-premises đến Amazon FSx for Windows File Server thông qua SMB.

VPC: (Virtual Private Cloud) 
- Dịch vụ cho phép tạo ra một mảng riêng với các thông số tùy chỉnh, bao gồm địa chỉ IP, subnet, và cổng kết nối Internet.
- 1 VPC có thể nằm trên nhiều AZ nhưng bắt buộc phải trong 1 region
- Trong 1 VPC thì tạo ra các subnet (VLAN) để allocate tài nguyên
- Giữa các VPC thì sử dụng VPC connection để kết nối với nhau
- Route table:
	- Dùng để xác định cách các gói tin được định tuyến (chuyển tiếp) giữa các tài nguyên trong VPC và ra/vào internet hoặc các mạng bên ngoài
	- Mỗi VPC có một bảng định tuyến chính mặc định, tự động gán cho tất cả các subnet mới nếu không có bảng định tuyến tùy chỉnh nào được chỉ định.
	- Bao gồm : Destination và Target
- Subnet :
	- Phân loại: (Do cấu hình route table)
		- Private: ko ra internet (hoặc phải NAT mới được ra internet)
		- Public: có thể ra internet trực tiếp
	- IP service:
		- Cấu trúc CIDR : định dạng <địa chỉ IP>/<số bit mạng>. Ví dụ: 192.168.1.0/24.
		- Public IP		: means the machine can be identified on the internet (WWW)
		- Private IP	: means the machine can only be identified on a private network only
		- Elastic IP 	: fixed public IP for instance, can be attached to one instance at a time	
	- Network Access Control Lists (ACLs): Là một danh sách các quy tắc bảo mật được áp dụng cho cả một subnet.
	- Security Groups: Là một tường lửa ảo cho các instance trong VPC, Điều chỉnh và kiểm soát lưu lượng truy cập vào và ra từ các instance EC2.
	- Use case:
		- ACL -> Security Groups -> EC2


- Traffic Mirroring – copy network traffic from ENIs for further analysis
- Route Tables – must be edited to add routes from subnets to the IGW, VPC Peering Connections, VPC Endpoints, …
- VPC Flow Logs 
	- VPC Flow Logs cho phép theo dõi lưu lượng mạng trong VPC, bao gồm các kết nối như RDP (cổng 3389) và SSH (cổng 22)
	– can be setup at the VPC / Subnet / ENI Level, for ACCEPT and REJECT traffic, helps identifying attacks, analyze using Athena or CloudWatch Logs Insights


Connection Insight VPC

		- Sử dụng cho các service lưu trữ cần cần hiệu năng cao
		- Aws service: Amazon S3 và DynamoDB.
		- Không hỗ trợ trực tiếp từ on-premises
	- Interface VPC Endpoint: 
		- Sử dụng cho các service khác
		- Aws service: AWS Lambda, Amazon RDS, AWS Secrets Manager, v.v.
		- Hỗ trợ trực tiếp từ on-premises
- ClassicLink 
	– connect EC2-Classic EC2 instances privately to your VPC


Connection to other VPC:
	- VPC Peering: kết nối thủ công giữa 2 VPC vs nhau ( hỗ trợ multiple Region)
	- Transit Gateway: 
		- Transit Gateway cung cấp khả năng kết nối tập trung giữa các VPC trong cùng một Region.
		- Transit Gateway Peering cho phép kết nối giữa các Transit Gateway ở các Region khác nhau,
	- AWS PrivateLink
		- Sử dụng khi kết nối 2 service giữa 2 VPC
		- Cho phép các dịch vụ (như ứng dụng hoặc API) trong một VPC có thể được truy cập từ các VPC khác thông qua endpoint riêng.
	- VPN/Direct Connect
		- Thường được sử dụng để tạo kết nối giữa on-premises vs AWS
		- Phức tạp khi sử dụng
			- Direct Connect gateway: Sử dụng cho các service lưu trữ cần cần hiệu năng cao
			- Direct Connect Interface: sử dụng cho các service khác
	- Virtual gateway/interface
		- Kết nối on-premises và AWS
		
		
Connection to Internet:
- Internet Gateway 
	- Cung cấp giải pháp kết nối Internet 2 chiều
	- Bao gồm cả IPv4 và IPv6
- Bastion Host
	– Máy Jump, sử dụng để ngoài internet kết nối vào, sau đó mới kết nới tới các private subnet 
	- public EC2 instance to SSH into, that has SSH connectivity to EC2 instances in private subnets
- NAT Instances 
	– Cho phép một máy chủ ở private subnet có thể kết nối ra internet
	- Phải quản lý thủ công, nhưng tích kiệm chi phí nếu dùng ít
	- Chỉ sử dụng cho IPv4
- NAT Gateway
	- Cung cấp giải pháp biến đổi (NAT) IP private thành 1 IP public để kết nối ra internet
	- Chỉ sử dụng cho IPv4
- Egress-only internet gateway 
	- For IPv6 targets
	- Giải pháp được thiết kế đặc biệt cho IPv6 để các EC2 instances trong private subnet có thể gửi lưu lượng ra ngoài internet 
	- Nhưng không bị kết nối ngược lại từ internet vào EC2 instances.

Route 53 (DNS)
- Quản lý và định tuyến lưu lượng mạng đến các ứng dụng đang chạy trên AWS,
- Cung cấp một số tính năng bổ trợ để tăng cường khả năng hoạt động và tính sẵn sàng của các ứng dụng.
- Các tính năng chính
	- DNS Management (Quản lý DNS)
	- Domain Registration (Đăng ký tên miền)
	- Health Checks and Monitoring (Kiểm tra sức khỏe và giám sát)
	- Traffic Flow and Routing Policies (Quy trình lưu lượng và chính sách định tuyến)
		- Simple routing: Chỉ định tuyến lưu lượng đến một tài nguyên.
		- Weighted routing: Phân phối lưu lượng đến các endpoint theo tỷ lệ (trọng số) xác định.
		- Latency routing: Chuyển hướng lưu lượng đến endpoint có độ trễ thấp nhất. -> high-performing 
		- Failover routing: Sử dụng endpoint dự phòng nếu endpoint chính không khả dụng.
		- Geolocation routing: Định tuyến dựa trên vị trí địa lý của người dùng.
			- Geoproximity routing policy: Phiên bản nâng cấp tốt hơn 
		- Multivalue answer routing: Trả về một danh sách các IP ngẫu nhiên để phân phối lưu lượng giữa các endpoint.
	- Scalability and High Availability (Khả năng mở rộng và tính sẵn sàng cao)
	- Cost-Effective (Hiệu quả về chi phí)
- Use case:
	- Sử dụng cho các ứng dụng thông thường, không cần hiệu năng quá cao:
		- Website Hosting: Định tuyến lưu lượng truy cập đến các ứng dụng web và trang web.
		- Hybrid Cloud Architectures: Kết nối các hệ thống trong đám mây và tại chỗ bằng cách định tuyến truy cập đến các endpoint trên AWS hoặc tại chỗ.
		- Disaster Recovery: Route 53 giúp định tuyến lưu lượng đến các endpoint dự phòng trong trường hợp xảy ra sự cố.
	- AWS <-> On-premises = Route 53 Resolver
		- Outbound Resolver = From your VPC (AWS) to On-premises or another VPC
		- Inbound Resolver = From on-premises network or another VPC TO your VPC.

1. Database Migration Service
-----------------------------------------------------------------------------
AWS DMS:
- Migrate data từ cơ sở dữ liệu nguồn sang cơ sở dữ liệu đích. 
- DMS thực hiện sao chép dữ liệu giữa các cơ sở dữ liệu đang chạy trên các nền tảng khác nhau mà không làm gián đoạn ứng dụng nguồn.
- Quickly and securely migrate databases to AWS, resilient, self healing
- The source database remains available during the migration
- Create an ongoing replication task.


AWS SCT:
- Chuyển đổi Metadata từ cơ sở dữ liệu nguồn sang cơ sở dữ liệu đích. 
- SCT giúp bạn chuyển đổi cấu trúc cơ sở dữ liệu (tables, indexes, triggers, stored procedures) từ hệ thống hiện tại sang hệ thống trên AWS.
- Convert your Database’s Schema from one engine to another
- Example OLTP: (SQL Server or Oracle) to MySQL, PostgreSQL, Aurora
- Example OLAP: (Teradata or Oracle) to Amazon Redshift
- Prefer compute-intensive instances to optimize data conversions
- You do not need to use SCT if you are migrating the same DB engine
	- Ex: On-Premise PostgreSQL => RDS PostgreSQL
	- The DB engine is still PostgreSQL (RDS is the platform) Source DB DMS + SCT Target DB (different engine)


AWS Snow Family 
- Là một tập hợp các dịch vụ và thiết bị lưu trữ và xử lý dữ liệu được thiết kế để giúp các tổ chức di chuyển, lưu trữ, và xử lý lượng dữ liệu lớn 
- Sử dụng trong môi trường không có kết nối Internet ổn định hoặc trong các địa điểm xa xôi.
- Chỉ tương thích trực tiếp với S3 hoặc phải qua EC2 làm trung gian
- Phân loại: 
	- AWS Snowcone device: ~8T
	- AWS Snowball Edge Storage Optimized device: ~20T
	- AWS Snowmobile vehicle : > 1000T
	- AWS Snowball Edge Compute Optimized with GPU device: Cần tính toán nặng
	

Amazon AppFlow
- Fully managed integration service that enables you to securely transfer data between Software-as-a-Service (SaaS) applications and AWS
- Sources: Salesforce, SAP, Zendesk, Slack, and ServiceNow
- Destinations: AWS services like Amazon S3, Amazon Redshift or nonAWS such as SnowFlake and Salesforce
- Frequency: on a schedule, in response to events, or on demand
- Data transformation capabilities like filtering and validation
- Encrypted over the public internet or privately over AWS PrivateLink
- Don’t spend time writing the integrations and leverage APIs immediately

AWS Transfer Family
- Serverless service: A fully-managed service for file transfers into and out of Amazon S3 or Amazon EFS using the FTP protocol
- Đa phần dành cho Khách hàng bên ngoài của công ty muốn tương tác vs hệ thống (ko phải KH của AWS)
	- KH của AWS thì dùng data sync
- Các chức năng chính:
	- Hỗ Trợ Nhiều Giao Thức: SFTP (SSH File Transfer Protocol), FTPS (FTP over SSL), và FTP, giúp dễ dàng tích hợp với nhiều hệ thống và ứng dụng khác nhau.
	- Tích Hợp Với Amazon S3: người dùng tải lên và tải xuống tệp trực tiếp từ Amazon S3, Điều này cho phép tổ chức sử dụng S3 như một hồ dữ liệu (data lake) để lưu trữ và quản lý tệp.

VIII. AWS Security & Encryption
----------------------------------------------------------------------------------------------------------------------------------------------------------
AWS KMS (Key Management Service)
- Dịch vụ quản lý key của AWS, sử dụng để mã hóa và giải mã dữ liệu trên AWS KMS
- Encryption keys is manage by AWS
- KMS Keys Types: KMS Customer Master Key
	- Symmetric (AES-256 keys)
		- Single encryption key that is used to Encrypt and Decrypt
		- AWS services that are integrated with KMS use Symmetric CMKs
		- You never get access to the KMS Key unencrypted (must call KMS API to use)
	- Asymmetric (RSA & ECC key pairs)
		- Public (Encrypt) and Private Key (Decrypt) pair
		- Used for Encrypt/Decrypt, or Sign/Verify operations
		- The public key is downloadable, but you can’t access the Private Key unencrypted
		- Use case: encryption outside of AWS by users who can’t call the KMS API
- Optional:
	- Use an AWS Key Management Service (AWS KMS) external key store backed by an external key manager
	- Create a customer managed key. Use the key to encrypt the EBS volumes
- Use case:
	- Encrypting and decrypting large datasets like certificates in near real-time.
	- Encryption outside of AWS by users who can’t call the KMS API
	- Apply a key policy to restrict key usage to a set of authorized principals.
	-  In each KMS key policy, deny decryption of data for all principals except an IAM role that the customer provides

- Deploy on
	- Application Load Balancer
	- API Gateway
	- CloudFront
	- AppSync GraphQL API
	- Cognito User Pool


AWS Shield: 
- Protect from DDoS attack (Distributed Denial of Service – many requests at the same time)
- AWS Shield Standard:
	- Free service that is activated for every AWS customer
	- Provides protection from attacks such as SYN/UDP Floods, Reflection attacks and other layer 3/layer 4 attacks
- AWS Shield Advance:
	- Cung cấp khả năng bảo vệ hệ thống trước các cuộc tấn công DDoS quy mô lớn. 
	- Gán ELB cho AWS Shield Advanced giúp bảo vệ kiến trúc của hệ thống với các tính năng bổ sung như phát hiện tấn công và phục hồi nhanh chóng.

	- WAF rules (Application Load Balancer, API Gateways, CloudFront)
	- AWS Shield Advanced (ALB, CLB, NLB, Elastic IP, CloudFront)
	- Security Groups for EC2, Application Load BAlancer and ENI resources in VPC
	- AWS Network Firewall (VPC Level)
	- Amazon Route 53 Resolver DNS Firewall
	- Policies are created at the region level
- Rules are applied to new resources as they are created (good for compliance) across all and future accounts in your Organization


Amazon GuardDuty
- Dịch vụ cung cấp cảnh báo các mối đe dọa và giám sát bảo mật của AWS
- Intelligent Threat discovery threat to protect your AWS Account
- Uses Machine Learning algorithms, anomaly detection, 3rd party data
- One click to enable (30 days trial), no need to install software
- Input data includes:
	- CloudTrail Events Logs – unusual API calls, unauthorized deployments
	- CloudTrail Management Events – create VPC subnet, create trail, …
	- CloudTrail S3 Data Events – get object, list objects, delete object, …
	- VPC Flow Logs – unusual internal traffic, unusual IP address
	- DNS Logs – compromised EC2 instances sending encoded data within DNS queries
	- Optional Features – EKS Audit Logs, RDS & Aurora, EBS, Lambda, S3 Data Events…
- Can protect against CryptoCurrency attacks (has a dedicated "finding" for it)


Amazon Inspector
- Dịch vụ bảo mật được cung cấp bởi AWS, giúp đánh giá và cải thiện mức độ bảo mật của ứng dụng đang chạy trên AWS. 
- Dịch vụ này tự động kiểm tra các tài nguyên AWS và cung cấp thông tin chi tiết về các lỗ hổng bảo mật, vấn đề cấu hình, và các yếu tố khác có thể ảnh hưởng đến bảo mật của ứng dụng.
- Các tính năng chính của Amazon Inspector:
	- Đánh giá lỗ hổng: Amazon Inspector quét các ứng dụng và máy chủ để phát hiện các lỗ hổng bảo mật và điểm yếu trong mã nguồn
	- Phân tích cấu hình: kiểm tra cấu hình của các tài nguyên AWS (như EC2 instances, ECS clusters, và RDS databases) để xác định các vấn đề có thể gây rủi ro bảo mật.
	- Báo cáo và đề xuất: Sau khi hoàn thành quét, Amazon Inspector cung cấp báo cáo chi tiết về các lỗ hổng và vấn đề bảo mật, cũng như các đề xuất cụ thể để khắc phục những vấn đề này.
	- Tích hợp với CI/CD: có thể được tích hợp vào quy trình CI/CD, cho phép kiểm tra bảo mật tự động trong suốt vòng đời phát triển phần mềm.
	- Dễ dàng sử dụng: giao diện đơn giản và dễ sử dụng, cho phép các nhà phát triển và đội ngũ bảo mật dễ dàng thiết lập và theo dõi các đánh giá bảo mật.
- Reporting & integration with AWS Security Hub
- Send findings to Amazon Event Bridge
